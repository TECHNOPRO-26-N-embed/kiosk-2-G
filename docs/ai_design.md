## 2. 要求仕様書（顧客ヒアリング結果）

### 2.1 顧客の要望（ヒアリングメモ）

* 商品を選択して購入できるようにしたい
* 投入金額を管理し、お釣りを返却できるようにしたい
* 在庫数を管理したい
* 売上データをCSVに保存したい
* 操作ログやバックアップ機能も用意したい

### 2.2 機能要件

| 機能ID | 機能名 | 概要 |
| :---: | :--- | :--- |
| F01 | 商品選択 | 商品番号や商品IDを入力して商品を選択する |
| F02 | 金額投入 | ユーザーから投入金額を受け取り、合計金額を管理する |

## 3. 基本設計

### 3.1 画面遷移図

```text
メインメニュー → [1] → 商品一覧表示 → [商品番号選択] → 金額投入 → 購入確認 → [確定] → 商品排出・お釣り返却 → メインメニュー
               → [2] → 投入金額返却 → メインメニュー
               → [3] → 管理メニュー → 在庫更新・商品入替・バックアップ
               → [4] → 終了
```

### 3.2 データの洗い出し

| 情報の名前 | 内容・説明 | 例 |
| :--- | :--- | :--- |
| 商品ID | 商品を識別する番号 | 1 |
| 商品名 | 販売する商品の名前 | 「水」 |
| 価格 | 商品の販売価格 | 120円 |
| 在庫数 | 現在の在庫数 | 10個 |
| 投入金額 | ユーザーが投入した金額 | 500円 |
| お釣り | 購入後に返却する金額 | 380円 |
| 売上金額 | 購入された商品の金額 | 120円 |
-------------------------------------------------------
| 操作ログ | 操作内容やエラー内容 | 「商品購入成功」 |

### 3.3 CSV保存形式

```csv
日時,商品ID,商品名,価格,在庫
2026-05-19 10:30:00,1,水,120,10
```

1. 商品選択・購入関連
select_product()
 商品を選択する処理。選択された商品IDや番号を受け取る。

purchase_product()
 選択された商品と投入金額をもとに購入処理を行う。購入可否判定も含む。

2. 金額管理・計算
insert_money()
 ユーザーからの投入金額を受け付け、合計金額を管理する。

calculate_change()
 購入後のお釣りを計算し、返却金額を算出する。

return_money()
 購入キャンセル時やお釣り返却時に、投入金額を返却する。

3. データ記録・ログ
record_sales_data()
 売上データをCSV等のファイルに保存する。

log_operation()
 操作履歴や障害予兆などのログを記録する。

4. 在庫管理
check_stock()
 指定商品の在庫数を確認する。

update_stock()
 購入や補充時に在庫数を更新する。

replace_product()
 商品入れ替えや新商品追加時に商品情報を更新する。

5. セキュリティ・保守
validate_operation()
 不正操作を検知・防止するためのチェック処理。

backup_data()
 障害発生時や保守用にデータをバックアップする。

6. その他
load_product_data()
 起動時に商品情報や在庫数をファイル等から読み込む。

save_product_data()
 終了時や在庫変更時に商品情報・在庫数を保存する。
