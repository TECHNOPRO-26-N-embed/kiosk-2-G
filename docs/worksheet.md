### 4.2 個別機能のアルゴリズム

> 各機能（購入処理、ログ保存、ログ表示など）の処理フローを記述してください。

#### 機能1: 商品選択　コ

```
struct Drink drinks[] と int choice、int money を受け取り、choice の数字を使って drinks[] から飲み物を選択します。
money が drinks.price より小さい場合は金額投入機能へ移動します。その後、代金計算機能へ移動します。
money が drinks.price より大きい場合は代金計算機能へ移動します。

金額投入機能では、投入したい金額を入力します。
```

#### 機能2:　代金計算 矢野（機能名）

```
選択された商品と投入された値段の計算 money -  price
計算結果をお釣りとして表示
```

#### 機能3:　売上計上 矢野（機能名）

```
単価*個数の計算　商品の数分　
計算結果を表示
```

#### 機能4:　csvファイルの作成・保存 佐藤（機能名）

```
ファイルを開く
ファイルに…
・購入データを書き込む
・「商品名,単価,数量,合計金額,日」を記入・保存
ファイルを閉じる
```

#### 機能5:　返却機能の実装 大橋（機能名）

```
硬貨が投入される際にどの硬貨かを識別し、カウントする(int count)。
商品が選択される、若しくはされている時にその商品の単価と投入金額に差異がある場合、差分(int dif)を導きだし、
その分の硬貨を排出する機能とする。
```

#### 機能6:　在庫管理機能 萩原（機能名）

```
商品が選択されたらint choiceに対応した飲み物のdrinks[]から１を引く
もしdrinks[]が0になったら在庫がないことを表示する
```
